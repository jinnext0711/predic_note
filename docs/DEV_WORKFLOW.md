# 開発ワークフロー

タスクを1つずつ進め、開発したものは **Claude Code で検証**し、バグや改善点があれば再開発する流れで進めます。

---

## 基本フロー

```
1. タスクを選ぶ（GitHub Issue または docs/DEV_TASKS.md）
       ↓
2. 開発する
       ↓
3. Claude Code で検証する（動作確認・コードレビュー・バグ・改善点の洗い出し）
       ↓
4. バグや改善点があれば → 再開発（2 に戻る）
       問題なければ → 次のタスクへ（1 に戻る）
```

---

## 検証で見るポイント（Claude Code に依頼するとき）

- **動作**: 期待どおり動くか
- **仕様**: MVP_SPEC・DEV_TASKS に沿っているか
- **バグ**: エラー・境界条件・例外処理
- **改善**: 可読性・保守性・パフォーマンス・テストのしやすさ

---

## 進め方の目安

- **Phase 1** から順に進める（データ基盤 → Scope 確定 → 導出指標）
- 1タスク完了ごとにコミット・プッシュし、Issue を Close する
- 検証で指摘された点は Issue または TODO で管理し、修正後に再度検証
