# predic_keiba 利用ガイド

中央競馬（JRA）に特化した「予想思考の構造化 × 過去検証可能な意思決定基盤」アプリの使い方です。

---

## 起動方法

```bash
# 仮想環境を有効化（初回のみ作成: python -m venv venv）
source venv/bin/activate

# アプリ起動
streamlit run src/app.py
```

---

## 全体の流れ

```
ユーザー登録 / ログイン
  ↓
① Scope（レース要件）を設定・保存
  ↓
② Must（除外条件）を設定・保存
  ↓
③ Prefer / Avoid（順位付け）を設定・保存
  ↓
④ （任意）カスタム変数を設定
  ↓
⑤ ロジック一覧で確認
  ↓
⑥-A シミュレーション可能 → バックテスト（有料）
⑥-B 全ロジック → フォワード成績を記録（無料）
```

---

## 1. ユーザー登録・ログイン

アプリにアクセスすると、まず認証画面が表示されます。

- **新規登録**: ユーザー名（3文字以上）とパスワード（6文字以上）を入力
- **ログイン**: 登録済みのユーザー名・パスワードでログイン
- ログアウトはサイドバーの「ログアウト」ボタンから

---

## 2. レース要件（Scope）

対象レースを定義する条件です。ロジック作成の最初のステップです。

1. 左メニューから「レース要件（Scope）」を選択
2. 以下の項目をそれぞれ1つ以上選択:
   - **競馬場**: 東京, 中山, 阪神, 京都, 中京, 小倉, 札幌, 函館, 福島, 新潟
   - **距離**: 短距離, マイル, 中距離, 長距離
   - **芝／ダート**: 芝, ダート
   - **クラス**: 未勝利, 1勝, 2勝, 3勝, オープン, G3, G2, G1
   - **年齢条件**: 2歳, 3歳, 3歳以上, 4歳以上
3. 「ロジック名」を入力（例: 東京芝マイル）
4. 「Scope をロジックに保存」をクリック

ルール:
- 各カテゴリ内はOR（東京 or 中山）
- カテゴリ間はAND（東京 or 中山 **かつ** 芝 **かつ** マイル）

---

## 3. 予想ロジック Must

ロジックが成立する最低条件です。満たさない馬は除外されます。

1. 左メニューから「予想ロジック Must」を選択
2. 対象ロジックを選択
3. ブロックごとに条件を追加:
   - **使用可能データ**: 前走着順, 前走4角位置, 前走距離, 斤量, 枠番／馬番, 最終オッズ帯, 血統指標
   - **演算子**: =, <=, >=, <, >, 含む
4. 「Must を保存」をクリック

ルール:
- 同一ブロック内はOR（条件A **または** 条件B を満たす）
- ブロック間はAND（ブロック1 **かつ** ブロック2 を満たす）
- 異なるカテゴリ間のORは禁止

---

## 4. Prefer / Avoid

Must を通過した馬の順位付けです。レキシコグラフィック方式（段階比較）で評価します。

### Prefer（優先条件）
- 最大5個まで設定可能
- 優先順位の並び替え可能（上にあるほど重要）
- 条件を満たす馬が上位に

### Avoid（回避条件）
- 最大2個まで設定可能
- 条件を満たす馬は順位が下がる

評価順序: まず Avoid に該当するかチェック → Prefer を優先順位1から順に段階比較

---

## 5. カスタム変数

ユーザー独自の変数を最大3個まで定義できます。

- **型**: 真偽値 / 3段階カテゴリ（高・中・低）/ 数値
- **注意**: カスタム変数を含むロジックは**シミュレーション不可**になります

---

## 6. ロジック一覧

保存済みロジックの確認・削除ができます。

- Scope / Must / Prefer-Avoid / カスタム変数の設定内容を一覧表示
- シミュレーション可否のバッジ表示
- ロジックの削除（確認ダイアログ付き）

---

## 7. フォワード成績（無料機能）

実運用での予想結果を記録・確認する機能です。全ロジックで利用可能です。

### 成績の記録
1. 左メニューから「フォワード成績」を選択
2. 対象ロジックを選択
3. フォームに入力:
   - レースID, レース日, レース名
   - 券種（単勝/複勝）
   - 予想馬名, 馬番
   - 賭け金（円）
   - 的中したか, 払戻金額
4. 「記録を保存」をクリック

### 成績の確認
- **サマリ**: 試行回数, 的中率, 回収率, 総損益
- **一覧**: 新しい順に表示。各記録の詳細表示・削除が可能

---

## 8. シミュレーション（有料機能）

シミュレーション可能ロジックに対して過去5年のバックテストを実行します。

### シミュレーション可能条件
- 完全選択式のみ（自由入力なし）
- 内部データ・導出指標のみ使用
- カスタム変数なし

### バックテスト
- **券種**: 単勝 / 複勝
- **賭け金**: 常に100円換算
- **オッズ**: 発走直前最終オッズに固定

### 出力項目
- 試行回数
- 回収率（%）
- 的中率（%）
- 年別推移
- 最大ドローダウン（円）
- 最大連敗数

---

## ロジック種別（二車線設計）

| 種別 | 条件 | 利用可能機能 |
|------|------|------------|
| シミュレーション可能 | 完全選択式・内部データのみ・カスタム変数なし | バックテスト + フォワード成績 |
| シミュレーション不可 | 自由入力/外部データ/カスタム変数あり | フォワード成績のみ |

---

## 導出指標（自動算出・定義固定）

ロジック内で参照可能な自動算出指標です。

| 指標 | 説明 |
|------|------|
| 逃げ馬数 | レース中の逃げ傾向馬の数（前走4角位置1〜2位） |
| 先行馬数 | レース中の先行傾向馬の数（前走4角位置3〜4位） |
| 前走位置平均との差 | 馬の前走4角位置とレース平均の差（正=後方寄り, 負=前方寄り） |
| レースペース簡易分類 | 速/普/遅（逃げ+先行馬の割合: >40%→速, <20%→遅, それ以外→普） |
