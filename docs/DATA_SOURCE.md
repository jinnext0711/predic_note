# データソース方針（タスク 1.1 関連）

## 対象範囲（MVP）

- **中央競馬（JRA）のみ**
- **過去5年分**
- **平地競走のみ**／**未勝利戦以上**／**新馬戦は除外**

---

## データソースの選択肢

| 候補 | 概要 | 備考 |
|------|------|------|
| JRA 公式 | 競馬場・レース結果等の公式発表 | 公式APIは一般公開されていない |
| netkeiba 等 | レース・馬・オッズ・結果の掲載サイト | スクレイピングは利用規約・法的注意が必要 |
| 有料データ提供 | 業者による整形データ | コスト・契約が必要 |
| 手動CSV等 | 自分で取得・整形したデータ | 小規模検証向け |

## 方針（現時点）

- **スキーマと取得・保存のパイプラインを先に定義**し、データソースは後から差し替え可能にする。
- 実装は **スタブ（モック）** から開始し、実際のデータ源が決まり次第、Fetcher を実装する。
- 保存形式: MVP では **JSON** または **SQLite** を想定（`data/` 以下に格納）。
